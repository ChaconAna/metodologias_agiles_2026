# Ejercicio Nro: 3
## Enunciado
¿Qué ventajas y desventajas tienen los diagramas UML?
## Resulocion 
•	Ventajas del UML
    o Lenguaje común: Es como un idioma universal que todos los programadores entienden, sin importar el lenguaje que usen. Ayuda a comunicarse mejor.
    o Sirve para planificar: Permite diseñar el sistema antes de programar, así se detectan errores más temprano.
    o Deja documentación: Los diagramas ayudan a entender el sistema sin tener que leer todo el código, sobre todo para gente nueva.
    o Ayuda a entender lo que pide el cliente: Con diagramas se puede ver qué va a hacer el sistema y confirmar si cumple con lo que el usuario quiere.
•	Desventajas del UML
    o No muestra todo el software: Es solo una simplificación, no puede representar toda la complejidad real.
    o Queda viejo rápido: Como el software cambia mucho, los diagramas se desactualizan si no se mantienen.
    o Puede hacer perder tiempo: Hacer y mantener muchos diagramas puede ser costoso y poco útil en proyectos cambiantes.
    o No asegura un buen resultado: Aunque los diagramas sean buenos, igual hay que probar el sistema con el usuario para saber si realmente sirve.
