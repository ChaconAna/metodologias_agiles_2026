# Ejercicio Nro: 7
## Enunciado
Dar un ejemplo de cada uno de los cuellos de botellas analizados anteriormente en el paper de Brooks
## Resulocion 
• Complejidad: Un sistema grande tiene muchas partes conectadas entre sí. Cuando cambias algo, puede afectar otras cosas sin que te des cuenta. Como nadie entiende todo el sistema completo, hacer cambios es difícil y arriesgado.
• Conformidad: El software muchas veces tiene que adaptarse a otros sistemas externos que no podés cambiar. Aunque sean complicados o estén mal diseñados, tenés que trabajar con ellos tal como son.
• Cambiabilidad: El software necesita cambiar todo el tiempo por factores externos: nuevas leyes, competencia o cambios tecnológicos. No es que esté mal hecho, sino que el entorno cambia constantemente.
• Invisibilidad: El software no se puede “ver” fácilmente como un plano físico. Si no hay buena documentación, entender cómo funciona puede llevar mucho tiempo antes de poder hacer cambios.
